Student Services Assistant (SSA) Chatbot

Emotion-Aware, Evidence-Grounded Student Support

An AI-powered chatbot designed to improve student services by detecting emotions, retrieving accurate answers from college-specific resources, and escalating unresolved queries to human advisors.

Team: Mandeep Singh, Kumari Nikitha Singh, Krishna

Problem Statement

Response Delays

Students face significant delays in getting answers to common queries, leading to frustration and decreased satisfaction with student services.

Lack of Empathy

Existing automated systems provide robotic responses without emotional intelligence, failing to address students' emotional states effectively.

Advisor Burden

Student advisors spend up to 60% of their time answering repetitive questions that could be automated, reducing capacity for complex cases.

Navigation Complexity

New students struggle to navigate complex policies and procedures across multiple departments, leading to confusion and misinformation.

Proposed Solution

A multi-stage intelligent chatbot system that combines emotion detection with hierarchical response generation to provide empathetic, accurate, and context-aware student support.

Core Features

  • Real-time emotion detection from text input
  • Intent classification for query routing
  • TF-IDF based FAQ retrieval for instant answers
  • LLM-based RAG fallback for complex queries
  • Seamless escalation to human advisors

Key Benefits

  • Empathy-first interaction design for distressed students
  • Instant response for common questions (90% coverage)
  • Context-aware follow-up and clarification
  • Reduced advisor workload by 40-60%
  • 24/7 availability with consistent quality

Key Components

  • Emotion Detection

    Using Gemini API to detect emotional tone (positive, negative, neutral) and trigger appropriate empathy templates.

  • Intent Classification

    Logistic Regression model categorizes queries into financial aid, academics, housing, and general services.

  • FAQ Retrieval

    TF-IDF vectorization with cosine similarity for matching queries to pre-approved institutional answers.

  • LLM Fallback

    For low-confidence matches, uses RAG with Gemini/OpenRouter to generate context-aware responses.

Decision Logic

Confidence Thresholds

> 0.85: Direct TF-IDF answer
0.65 - 0.85: Answer with verification prompt
< 0.65: LLM RAG fallback

Emotion Handling

Negative: Empathy template + advisor offer
Neutral/Positive: Direct query handling

UX & Empathy System

Emotion Handling

Negative Emotions

Triggers empathy templates and offers to connect with human advisor. Response includes validation of feelings before addressing query.

Positive/Neutral Tone

Direct query handling with optional friendly tone markers. Maintains efficiency while being approachable.

Conversation Design

Context Retention

Maintains conversation context for 10 minutes of inactivity. Optional history toggle for longer retention.

Follow-up Prompts

Dynamic suggestions like "Would you like to know more?" or "Should I connect you with an advisor?" based on query complexity.

Deployment & Operations

Implementation Details

  • Conversation Flow

    Shows only latest reply by default with expandable history toggle to reduce cognitive load.

    Architecture designed for easy translation layer addition to support diverse student populations.
